# Day 2 - Salesforce Platform Basics

## 1. What is Salesforce Platform?

Salesforce Platform is a cloud-based platform used to build and customize business applications. It helps organizations manage customer data, automate business processes, create apps, generate reports, and improve productivity.

The platform supports:
- CRM functionality
- App development
- Automation
- AI features
- Integration with external systems

It provides both:
- No-code/low-code tools
- Coding options using Apex and Lightning Web Components

<hr>

# 2. Salesforce Concepts

## App

An App in Salesforce is a collection of objects, tabs, tools, and features designed for a specific business purpose.

### Examples
- Sales App
- Service App
- College Management App

Apps help users work in one organized environment.



## Object

An Object is a database table that stores data in Salesforce.

Objects contain:
- Records → rows
- Fields → columns

### Examples
- Account Object
- Contact Object
- Student Object

Objects can be:
- Standard Objects
- Custom Objects



## Tab

A Tab is a user interface element used to open objects, pages, or features quickly.

### Examples
- Contacts Tab
- Accounts Tab
- Students Tab

Tabs make navigation easier for users.

# 3. Difference: Configuration vs Coding

## Configuration (No Code)
Configuration means customizing Salesforce using drag-and-drop tools without writing code.

### Features
- No coding knowledge required
- Faster and easier to implement
- Used for simple automation and customization
- Uses tools like Flow Builder, Validation Rules, and Page Layouts

### Examples
1. Creating a custom field on the Contact object  
2. Sending automatic emails using Flow Builder  



## Coding (Apex)
Coding means developing custom functionality using programming languages like Apex and Lightning Web Components.

### Features
- Requires programming knowledge
- Used for advanced business logic
- Supports complex integrations and automation
- Provides more flexibility and control

### Examples
1. Integrating Salesforce with external payment systems  
2. Creating advanced validation logic using Apex
